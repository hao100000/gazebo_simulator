# 24.04だけでなく22.04にも対応

このリポジトリの `gazebo_simulator` は、もともと Ubuntu 24.04 / ROS 2 Jazzy / Gazebo Sim 系を前提にしていた。  
そこに加えて、Ubuntu 22.04 / ROS 2 Humble / Gazebo Sim 6 (`ign gazebo`) でも動くように整理した。

## 変更の要点

### 24.04 向け

- `launch/gazebo_24.04.launch.py` は既存の Gazebo Sim 系の起動構成を維持した
- `gz sim` 前提、`ros_gz_bridge` 前提、`gz_ros2_control` 前提の流れはそのまま
- モデル生成は `xacro` と `gz sdf -p` を使う
- `can_to_gazebo` と `teleop_CANArray_keyboard` をまとめて起動する

### 22.04 向け

- `launch/gazebo_22.04.launch.py` を追加した
- `ign gazebo -r` を使って Gazebo Sim 6 を起動する
- `xacro` で URDF を生成し、`ign sdf -p` で SDF を再生成する
- `IGN_GAZEBO_RESOURCE_PATH` に `src/gazebo_simulator/models` を足す
- `IGN_GAZEBO_SYSTEM_PLUGIN_PATH` に Gazebo Sim 6 の plugin path と ROS 側の lib を足す
- `ros_gz_bridge` が無いときは、Humble 用の apt パッケージを入れるようにエラーを出す
- `gz_ros2_control` が無いときも同様に止める
- `teleop_CANArray_keyboard` は `xterm` がある環境でだけ起動するようにした
- `enable_teleop` 引数を追加して、キーボード操作の有無を切り替えられるようにした

## 修正した理由

- 22.04 側では `ROS2_WS` が相対パスだと、`xacro` に存在しないパスが渡って失敗した
- `can_to_gazebo.cpp` は `ROS2_WS` を見て `config/motors_*.yaml` を読むため、launch 側で絶対パスに正規化する必要があった
- `xterm` が無い環境でも launch 全体が落ちないようにしたかった
- 22.04 の Humble 環境では `ros_gz_bridge` と `gz_ros2_control` を明示的に入れる必要がある

## 確認できたこと

- `xacro` の実行は通った
- `ign sdf -p` による SDF 生成は通った
- `can_to_gazebo` は `motors_arm_2_v10.yaml` を正しく読めた
- `ros2 launch gazebo_simulator gazebo_22.04.launch.py` は、今回の修正前に出ていた `xacro` パスエラーと `xterm` エラーを解消できた

## 起動メモ

### 24.04

```bash
ros2 launch gazebo_simulator gazebo_24.04.launch.py
```

### 22.04

```bash
ros2 launch gazebo_simulator gazebo_22.04.launch.py
```

## 補足

- 22.04 側は Gazebo Classic ではなく Gazebo Sim 6 を使う
- そのため、bridge と ros2_control は Humble 向けの Gazebo Sim 系パッケージが前提になる
- `ROS2_WS` は絶対パスにしておくのが安全
